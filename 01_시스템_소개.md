# SCM + P&L 운영 분석 시스템 소개

> **이 문서를 읽는 분께:** 개발 지식 없이도 읽을 수 있도록 작성했습니다.
> 기술 세부사항이 필요하시면 `docs/` 폴더의 원본 문서를 참고하세요.

---

## 이 시스템이 왜 필요한가요?

여러 곳에 흩어진 데이터(쇼핑몰 주문, 물류센터 출고, 창고 재고, 재무 비용 등)를 **한 곳에 모아** 자동으로 분석해주는 시스템입니다.

**도입 전 상황:**
- 재고 현황 → 창고 엑셀
- 주문 현황 → 쇼핑몰 관리자
- 손익 현황 → 재무팀 스프레드시트
- 비용 내역 → 3PL 인보이스

각각 따로 보다 보니 전체 그림이 안 보이고, 데이터를 취합하는 데만 시간이 걸렸습니다.

**도입 후:**
- 모든 데이터가 한 곳에 자동 집계
- 웹 대시보드에서 즉시 조회
- 이상 징후 자동 감지 및 경고

---

## 데이터는 어떻게 흐르나요?

```
  [원본 데이터]          [자동 처리]          [결과]

  쇼핑몰 주문 파일  ──┐
  물류센터 출고 파일 ──┤                    ┌── SCM 대시보드
  창고 재고 파일  ────┤──▶ 시스템이 자동 ──▶│   (재고/발주/출고)
  구매발주 파일   ────┤    정리·분석         │
  재무 비용 파일  ────┤                    └── P&L 대시보드
  정산 인보이스  ─────┘                        (매출/비용/손익)

  ↑ 담당자가 파일을 넣기만 하면 됩니다
```

**할 일은 단 2가지:**
1. 파일을 `inbox/` 폴더에 넣는다
2. 명령어 하나(`python run.py --once`)를 실행한다

## 누가 무엇을 보나요?

| 역할 | 주로 보는 화면 | 주요 확인 사항 |
|------|--------------|--------------|
| **운영팀 실무자** | SCM 대시보드 | 재고현황, 품절 위험, 납기 지연 |
| **SCM 기획팀** | SCM 대시보드 | 공급망 병목, 서비스레벨, 발주 현황 |
| **재무/회계팀** | P&L 대시보드 | 매출·원가·손익, 정산 검증 |
| **팀장/경영진** | 두 대시보드 모두 | 핵심 KPI 요약, 수익성 순위 |

---

## 대시보드는 어떻게 여나요?

**SCM 대시보드** (재고·발주·출고 분석):
- 주소창에 입력: `http://localhost:8501`
- 재고, 품절위험, 발주현황, 서비스레벨 등 11개 탭

**P&L 대시보드** (매출·비용·손익 분석):
- 주소창에 입력: `http://localhost:8502`
- 매출, 원가, 마진, 손익 등 11개 탭

> 대시보드를 열려면 먼저 담당자가 시스템을 실행해 놓아야 합니다.

## 핵심 용어 설명

어렵게 느껴지는 용어를 쉬운 말로 풀어드립니다.

| 용어 | 쉬운 설명 |
|------|----------|
| **데이터 파이프라인** | "파일 넣으면 자동으로 정리해주는 과정" |
| **RAW → CORE → MART** | "원본 → 정제 → 분석용"으로 데이터가 단계별로 가공됨 |
| **KPI** | 핵심 성과 지표. 재고일수, 서비스레벨, 마진율 같은 수치들 |
| **커버리지** | 분석에 필요한 데이터가 얼마나 채워져 있는지 (100%가 이상적) |
| **기간 마감** | 한 달치 데이터를 확정하는 월말 결산 작업 |
| **DQ (데이터 품질) 검사** | 파일에 오류가 없는지 자동으로 확인하는 과정 |
| **ACTUAL / PARTIAL** | 데이터가 완전히 확인됨 / 일부 누락이 있어 추정값 포함 |
| **대사 (검증)** | 두 시스템의 숫자가 일치하는지 교차 확인하는 작업 |
| **SCM** | Supply Chain Management. 주문부터 배송까지 공급망 전체 |
| **P&L** | Profit & Loss. 손익계산서 (매출 - 비용 = 이익) |
| **3PL** | 외부 물류 업체 (창고 운영, 출고 등을 대행) |

## 데이터 종류 한눈에 보기

| 데이터 | 발생 주기 | 누가 주나 |
|--------|----------|----------|
| 주문 내역 | 매일 | 쇼핑몰/OMS |
| 출고 내역 | 매일 | 물류센터/WMS |
| 재고 현황 | 매일 | 물류센터/WMS |
| 구매발주 | 수시 (발주할 때) | 구매팀 |
| 입고 내역 | 수시 (도착할 때) | 물류센터 |
| 반품 내역 | 수시 (반품 접수 시) | CS팀 |
| 상품 마스터 | 수시 (신상품/변경 시) | 상품기획팀 |
| 정산 인보이스 | 월 1~2회 | 3PL/운송사 |
| 비용 데이터 | 월 1회 (마감 전) | 재무팀 |
| 환율 | 월 1회 (마감 전) | 재무팀 |

> 자세한 데이터 입력 방법은 [02_데이터_입력_안내.md](02_데이터_입력_안내.md)를 참고하세요.

## 자주 받는 질문

**Q. 매일 파일을 꼭 넣어야 하나요?**
꼭 매일일 필요는 없습니다. 며칠치를 모아서 한번에 넣어도 됩니다. 다만 재고·품절 분석은 최신 데이터일수록 정확합니다.

**Q. 파일을 잘못 넣으면 어떻게 되나요?**
시스템이 자동으로 오류를 감지하고 거부합니다. 잘못된 파일은 데이터베이스에 저장되지 않으므로 안전합니다.

**Q. 손익 분석은 언제부터 볼 수 있나요?**
매일 넣는 재고·주문·출고 데이터만으로는 손익 분석이 완전하지 않습니다. 월말에 정산·비용·환율 데이터를 추가로 넣어야 정확한 손익 수치를 볼 수 있습니다.

**Q. 기술 담당자가 없으면 못 쓰나요?**
파일 넣기와 명령어 실행은 비개발자도 할 수 있습니다. 초기 설치와 오류 발생 시에는 기술 담당자 도움이 필요할 수 있습니다.
