# 이상징후 대응 가이드

> **대상:** 운영팀, SCM 기획팀, 재무팀
> 대시보드에서 경고가 보일 때 어떻게 해석하고 대응할지 안내합니다.

---

## 경고 신호 등급

| 등급 | 표시 | 의미 | 필요한 행동 |
|------|-----|-----|----------|
| **CRITICAL** | 🔴 빨강 | 즉각 대응 필요. 방치하면 심각한 운영 차질 | 당일 대응 |
| **HIGH** | 🟠 주황 | 빠른 대응 필요. 48시간 내 해결 권장 | 2일 내 대응 |
| **MEDIUM** | 🟡 노랑 | 주의 요망. 주간 검토 시 확인 | 주간 검토 |
| **LOW** | 🔵 파랑 | 정보성. 참고만 하면 됨 | 월간 검토 |

---

## 영역별 경고 신호와 대응

### 1. 공급 문제

#### 발주 지연율 20% 이상 (HIGH/CRITICAL)
**의미:** 공급업체 10곳 중 2곳 이상이 납기를 지키지 못하고 있음

**원인:** 공급업체 생산 지연, 재료 수급 문제, 운송 차질

**대응 행동:**
- 구매팀에 해당 공급업체 즉시 연락 요청
- 납기 지연이 지속되는 업체는 대체 업체 발굴 검토
- 영향받는 품목의 안전재고 수준 임시 상향
- 긴급 배송(항공 등) 필요 여부 검토

---

#### 공급업체 납기 7일 이상 초과 (HIGH/CRITICAL)
**의미:** 특정 공급업체 물건이 예정보다 1주일 이상 늦어지고 있음

**대응 행동:**
- 해당 공급업체에 원인 및 도착 예정일 확인
- 해당 품목 품절 위험 점검 (품절 위험 탭 확인)
- 대체 공급업체 또는 대체 상품 긴급 검토

---

### 2. 창고/물류센터 문제

#### 미처리 주문(백로그) 200건 이상 (HIGH/CRITICAL)
**의미:** 출고해야 하는데 아직 처리하지 못한 주문이 200건을 넘음

**원인:** 물류센터 처리 인력 부족, 피킹 효율 저하, 시스템 오류

**대응 행동:**
- 3PL 담당자에게 즉시 연락하여 현황 파악
- 임시 인력 투입 또는 야간 작업 검토
- 납기 기준 우선순위 재정렬 (긴급 주문 먼저)
- 고객 알림 필요 여부 검토

---

#### 처리량 부족 (피킹·패킹 효율 80% 미만, HIGH)
**의미:** 물류센터의 실제 처리 속도가 수요를 따라가지 못함

**대응 행동:**
- 3PL과 프로세스 개선 방안 협의
- 피크 기간 인력 계획 재검토
- 설비·장비 투자 필요성 검토

---

### 3. 물류/통관 문제

#### 통관 지연 5일 이상 (HIGH)
**의미:** 수입 화물이 통관에서 5일 이상 묶여 있음

**원인:** 서류 미비, 세관 검사, 수입 규제 이슈

**대응 행동:**
- 관세사/포워더에 즉시 연락하여 원인 파악
- 부족한 서류 긴급 준비
- 영향받는 품목 재고 현황 확인 (품절 위험 탭)
- 유사한 상품을 국내 재고에서 대체 출고 가능한지 검토

---

#### 운송사 정시 배송율 90% 미만 (HIGH)
**의미:** 10건 중 1건 이상이 배송 약속일을 지키지 못하고 있음

**대응 행동:**
- 해당 운송사와 원인 및 개선 계획 협의
- 반복 미달 시 운송사 변경 검토
- SLA(서비스 수준 계약) 위반에 따른 페널티 적용 검토

---

### 4. 수요/채널 문제

#### 반품율 급증 (기준 대비 1.5배 이상, HIGH)
**의미:** 갑자기 반품이 평소보다 50% 이상 많아짐

**원인:** 품질 불량, 오출고, 배송 파손, 상품 설명 불일치

**대응 행동:**
- 반품 분석 탭에서 사유 확인 (불량? 오출고? 단순 변심?)
- 불량/오출고가 원인이면 즉시 품질 조사 실시
- 해당 품목 출고 일시 중단 검토 (품질 확인 전까지)
- CS팀과 대응 메시지 협의

---

#### 프로모션 수요 폭증 (예측 대비 2배 이상, HIGH)
**의미:** 프로모션 효과가 예상보다 훨씬 커서 재고/처리 용량 초과 위험

**대응 행동:**
- 재고 현황 즉시 확인 (품절 위험 탭)
- 물류센터 처리 용량 긴급 점검
- 주문 마감/대기자 명단 운영 필요 여부 검토
- 다음 프로모션에는 사전 재고 배치 계획 수립

---

### 5. 재무 문제

#### 유통기한 초과 재고 발생 (CRITICAL)
**의미:** 유통기한이 지난 재고가 있음 → 즉시 처분 필요

**대응 행동:**
- 유통기한 관리 탭에서 해당 품목·수량 확인
- 즉시 폐기 처리 (식품 등 규제 대상이면 규정에 따라 처리)
- 할인 판매 또는 기부 가능 여부 검토 (유통기한 임박 재고)
- 재발 방지: FEFO 피킹 절차 준수 여부 확인
- 재발주 계획 재검토 (과다 발주가 원인인지 확인)

---

#### 과재고 금액 임계값 초과 (HIGH)
**의미:** 재고에 너무 많은 자금이 묶여 있음

**대응 행동:**
- 과재고 탭에서 금액이 큰 품목 확인
- 해당 품목 재발주 즉시 중단
- 판촉 이벤트, 번들 판매, 할인 판매 검토
- 유통기한이 있는 경우 폐기 위험금액 추가 확인

---

## 데이터 품질 경고

시스템이 파일을 처리할 때 발견한 오류입니다. 대시보드의 상태 화면 또는 `python run.py --status` 결과에서 확인 가능합니다.

### 파일이 완전히 거부된 경우 (CRITICAL/HIGH 오류)

| 경고 메시지 | 원인 | 해결 방법 |
|-----------|-----|---------|
| 필수 컬럼 누락 | 파일에 필수 항목이 없음 | 빠진 컬럼 추가 후 재업로드 |
| 주문번호에 빈 값 | 고유 식별자에 빈 셀 존재 | 빈 셀 채운 후 재업로드 |
| 중복 주문번호 | 같은 건이 두 번 포함됨 | 중복 행 제거 후 재업로드 |
| 알 수 없는 비용 코드 | 새로운 비용 항목이 시스템에 미등록 | 개발팀에 새 코드 등록 요청 |
| 날짜 형식 오류 | 날짜가 잘못된 형식 (예: 250115 대신 20250115) | 날짜 형식 수정 후 재업로드 |
| 유통기한 누락 | 유통기한 관리 품목에 유통기한 미기재 | 유통기한 입력 후 재업로드 |

### 경고만 기록되고 파일은 저장된 경우 (MEDIUM/LOW 오류)

| 경고 메시지 | 의미 | 확인 방법 |
|-----------|-----|---------|
| 상품 마스터 미등록 품목 | 파일에 있는 상품코드가 상품 마스터에 없음 | 상품 마스터 파일 업로드 또는 개발팀에 등록 요청 |
| 수량 음수 | 수량 값이 마이너스 | 원본 데이터 확인 (반품이라면 정상일 수 있음) |
| 이상 금액 | 금액이 평균의 10배 초과 | 입력 오류인지 확인 (0 하나 더 붙은 경우 등) |
| 날짜 범위 초과 | 2020년 이전 또는 1년 후보다 먼 날짜 | 날짜 입력 오류 확인 |

## 대사(검증) 불일치 발생 시

대사/검증 탭에서 **FAIL** 또는 **WARN** 상태가 보일 때:

| 대사 항목 | FAIL 의미 | 조치 |
|---------|---------|-----|
| 재고 수불 대사 | 기초 + 입고 - 출고 ≠ 기말 재고 | 특정 날짜 데이터 누락 또는 오류 확인 |
| OMS vs WMS 대사 | 쇼핑몰 출고 건수 ≠ 물류센터 출고 건수 | 미매칭 건수 확인, 양쪽 시스템 대조 |
| 정산 대사 | 3PL 청구액 ≠ 시스템 계산액 | 청구서 원본 재확인, 단가표 확인 |

> 대사 불일치는 데이터 오류인 경우도 있지만, 실제 업무 오류(오출고, 청구 오류 등)를 나타낼 수도 있습니다.

## 에스컬레이션 기준

| 상황 | 담당자 | 시간 기준 |
|------|--------|---------|
| CRITICAL 경고 발생 | 팀장 즉시 보고 | 당일 |
| 품절 발생 | 구매팀 + 팀장 | 당일 |
| 반품율 급증 | CS팀 + 품질팀 + 팀장 | 당일 |
| 정산 대사 FAIL | 재무팀 + 3PL 담당자 | 2일 내 |
| 데이터 파일 오류 반복 | 개발팀 | 1주일 내 |
